# PRD: サボれない習慣化アプリ

## 1. コンセプト

「習慣をサボったら、自分のXアカウントに恥ずかしい投稿が自動で行われる」という強制力によって習慣化を実現するアプリ。従来の習慣化アプリは「ごほうび」型だが、本アプリは「ペナルティ」型で差別化する。

## 2. ターゲットユーザー

- 20〜30代のXユーザー
- 習慣化に何度も挫折した経験がある人
- 「自分に甘い」と自覚しており、外圧が欲しい人
- Xのフォロワーに見られたくないという心理的圧力が効く人

## 3. コア機能詳細

### 3.1 習慣登録

- **タスク名**: 自由入力（例: 「筋トレ30分」「英語学習」「読書」）
- **期限時刻**: 時間を設定（例: 21:00）。この時刻までに「完了」しなければペナルティ発動
- **繰り返し**: 曜日選択（月〜日のマルチセレクト）
- **スロット制限**:
  - 無料ユーザー: 1個まで
  - Proユーザー: 10個まで
- 習慣の編集・削除は設定画面から可能
- 削除時は確認ダイアログを表示

### 3.2 完了判定

- ホーム画面のタスクカードに「完了」ボタンを配置
- タップ後に確認ダイアログ: 「本当に完了しましたか？」→「はい」で確定
- 完了時刻をDBに記録
- 完了済みタスクはカードの見た目が変わる（チェックマーク + グレーアウト）
- **期限時刻を過ぎたタスクは「完了」ボタンを無効化**（ペナルティ確定）

### 3.3 ペナルティ投稿

#### トリガー
- 期限時刻到達時にSupabase Edge Functionが未完了を検知
- **即座にXへ投稿**（猶予なし）

#### ペナルティタイプ（ユーザーが初回設定時に選択）
1. **テキストモード**
   - 固定文: `私はサボりました。だらしのない人間です。 #サボれない習慣化アプリ`
   - ユーザーはペナルティ文のカスタマイズ不可（MVP）
2. **自撮りモード**
   - ユーザーが**事前に**インカメラで自撮りを撮影・保存
   - 撮影した写真はSupabase Storageに暗号化保存
   - ペナルティ発動時: 保存済み自撮り + テキストを合わせてXに投稿
   - テキスト: `[自撮り写真] 私はサボりました。だらしのない人間です。 #サボれない習慣化アプリ`

#### 投稿後の処理
- アプリ内プッシュ通知: 「ペナルティが執行されました」
- ホーム画面にペナルティ履歴バッジを表示
- ストリークをリセット

### 3.4 X OAuth連携

- **認証方式**: OAuth 2.0 PKCE (Authorization Code Flow with PKCE)
- **必要スコープ**: `tweet.read`, `tweet.write`, `users.read`, `offline.access`
- `offline.access`でRefresh Tokenを取得し、バックグラウンド投稿を可能にする
- Refresh Tokenの保存先: Supabase DB（暗号化カラム）
- Token期限切れ時の自動リフレッシュ処理が必要

### 3.5 プッシュ通知

- **リマインダー通知**: 期限の30分前に「まだ完了していません！残り30分です」
- **ペナルティ通知**: 「ペナルティが執行されました。Xを確認してください」
- Expo Notifications APIを使用
- 通知許可のリクエストはオンボーディング完了後

### 3.6 ストリーク表示

- ホーム画面上部に連続達成日数を大きく表示
- 炎アイコン + 日数カウンター
- ペナルティ発動時にリセット
- 最長ストリーク記録も保持

## 4. マネタイズ

### 4.1 プラン構成

| | Free | Pro（¥300/月） |
|--|------|---------------|
| 習慣スロット | 1個 | 10個 |
| バナー広告 | 表示あり | 非表示 |
| ペナルティ投稿 | ○ | ○ |
| ストリーク表示 | ○ | ○ |

### 4.2 広告

- **SDK**: Google AdMob (`react-native-google-mobile-ads`)
- **フォーマット**: バナー広告（ホーム画面下部に常時表示）
- **Proユーザー**: 広告非表示
- AdMobアカウント・アプリIDはApp Store申請前に取得が必要

### 4.3 課金

- **SDK**: RevenueCat (`react-native-purchases`)
- App Store In-App Purchase（自動更新サブスクリプション）
- RevenueCatがレシート検証・ステータス管理を担当
- 課金状態はRevenueCat → Supabase webhookで同期

## 5. 画面一覧

### 5.1 オンボーディング（3スライド）

| スライド | 内容 |
|---------|------|
| 1 | コンセプト説明: 「サボったら自動投稿される習慣化アプリ」+ イラスト |
| 2 | ペナルティ例の表示: テキスト投稿と自撮り投稿のモックアップ |
| 3 | 免責事項 + 「同意して始める」ボタン |

### 5.2 ログイン画面

- 「Xアカウントでログイン」ボタン1つ
- OAuth 2.0 PKCEフロー開始
- ログイン完了後 → ペナルティ設定画面へ遷移

### 5.3 ペナルティ設定画面

- ペナルティタイプ選択: 「テキストのみ」 / 「自撮り写真つき」
- 自撮り選択時: カメラ起動ボタン → インカメラで撮影 → プレビュー → 保存
- 「この設定で始める」ボタン → 習慣追加画面へ

### 5.4 ホーム画面（タブ1）

- 上部: ストリーク表示（炎アイコン + 連続日数）
- 中央: 今日のタスクカード一覧
  - 各カード: タスク名 / 期限時刻 / カウントダウン / 完了ボタン
  - 完了済み: チェックマーク + グレーアウト
  - 期限切れ: 赤枠 + 「ペナルティ執行済み」ラベル
- 下部: バナー広告（Freeユーザーのみ）
- FAB（+ボタン）: 習慣追加モーダルを開く
  - 無料ユーザーが1個登録済みの場合 → Proアップグレード誘導

### 5.5 履歴画面（タブ2）

- カレンダービュー（月間）
  - 達成日: 緑マーク
  - ペナルティ日: 赤マーク
  - 未設定日: グレー
- 下部: 月間達成率（%表示）

### 5.6 設定画面（タブ3）

- Xアカウント情報（アイコン・ユーザー名）
- ペナルティ設定の変更
- 習慣の編集・削除
- Proプランへのアップグレード / 管理
- 通知設定
- ログアウト
- アカウント削除

### 5.7 習慣追加モーダル

- タスク名入力
- 期限時刻ピッカー
- 曜日選択（マルチセレクト）
- 「追加」ボタン

### 5.8 自撮り撮影画面

- インカメラプレビュー
- 撮影ボタン
- 撮影後: プレビュー + 「この写真を使う」 / 「撮り直す」
